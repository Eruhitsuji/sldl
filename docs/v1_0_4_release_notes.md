# SLDL v1.0.4 Release Notes

v1.0.4 is a small maintenance release focused on template manifest policy, generated template references, and stronger build-manifest traceability.

## Highlights

- `templates/template_manifest.json` is now the canonical bundled template manifest.
- `templates/manifest.json` remains as a legacy compatibility copy and declares `manifest_role: "legacy_compatibility"`.
- `template docs --format markdown|json` generates a template reference from the active manifest.
- Project files generated by `template project` record `template_manifest_role`.
- Build manifests produced by `project build` preserve template name, source, manifest, manifest role, and declared document type.
- `quality manifest` validates template metadata and rejects template build manifests that point to the legacy manifest instead of the canonical manifest.
- `docs/project_workflow.md` and `docs/commands_reference.md` are synchronized with the v1.0.4 template workflow.

## Compatibility

Existing v1.0.1-v1.0.3 template workflows continue to work. The preferred edit target is now explicit: edit `templates/template_manifest.json` first, then refresh `templates/manifest.json` only as a compatibility copy.

## Recommended checks

```bash
python3 -m pytest -q
python3 -S -m sldl_compiler.cli template docs --format markdown -o docs/generated_template_reference.md
python3 -S -m sldl_compiler.cli quality release \
  --targets examples/release_check.json \
  --manifest build/release_manifest.json
```
