# Version reset policy for v1.0

v0.15.2 reset pre-public compatibility baggage. v1.0.0 adds the bilingual documentation and official sample baseline intended for v1.0.0.

## v1.0 distribution rule

The v1.0.0 distribution should contain:

- current source code
- current tests
- current templates
- current user-facing documentation
- current official bilingual examples

The v1.0.0 distribution should not contain:

- historical generated outputs
- old release manifests
- old version-numbered sample files
- old per-version development notes

## Rationale

The earlier v0 line was unpublished and experimental. Keeping every experimental sample would make the first public version harder to understand and harder to document. The v1.0 package should therefore start from a clean, stable baseline.
